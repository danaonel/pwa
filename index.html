Progressive Web Apps (PWA)

Progressive web apps are technically regular web pages (or websites) but can appear to the user like traditional applications or (native) mobile applications.

Benefits

PWA vs Websites

  • App-like appearance
  • Connectivity-independent
  • Re-engageable

PWA vs Native apps

  • Discoverable to search engines
  • Linkable (can use URI to reload the PWA at the current state)

Requirements

Mandatory Requirements

An app shell that is:

  • Progressive
  • Responsive
  • Small/Fast
  • Meta tags

Mandatory Requirements(cont)

  • A web manifest
  • A service worker
  • HTTPS/Firebase

Optional Requirements

  • Icons for all screens
  • A tool such as Lighthouse to measure progress

The app

SPAs are perfect for this purpose.

The Web Manifest

The W3C Web App manifest is a JSON file that makes your app indexable by search engines not as a usual website, but as a web app.

Adds a splash screen.

Makes the app installable without any app stores.

The Service Worker

The Service Worker is a plain JavaScript file which runs in the background and provides the app with the ability to work offline.

It uses Events actions and wake-ups and works only if it’s needed. It responds to the events, including network requests.

It works in conjunction with the Cache API.

Things to remember:

  1. Check if browser supports service workers before registering it
  2. The service worker scope is defined by the directory where it has been placed
